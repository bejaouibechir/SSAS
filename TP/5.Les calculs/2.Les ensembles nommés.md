## Partie 1 : Comprendre les Named Sets

### Différence Statique vs Dynamique

| Type          | Évaluation            | Utilisation           | Performance   |
| ------------- | --------------------- | --------------------- | ------------- |
| **Statique**  | Au traitement du cube | Liste fixe de membres | Meilleure     |
| **Dynamique** | À chaque requête      | S'adapte au contexte  | Plus flexible |

### Quand utiliser quel type ?

**Statique** :

- Listes de produits spécifiques (promotions)
- Périodes fiscales fixes
- Segments prédéfinis

**Dynamique** :

- Top N produits (change selon le contexte)
- Périodes relatives (6 derniers mois)
- Filtres adaptatifs

## Partie 3 : Named Set 1 - Top 10 Produits (Dynamique)

### Contexte Métier

Identifier les 10 produits générant le plus de chiffre d'affaires, quel que soit le contexte d'analyse (période, région, canal).

### Création

1. **New Named Set**

2. **Name** : `[Top 10 Produits CA]`

3. **Type** : **Dynamic**

4. **Expression MDX** :

```mdx
TOPCOUNT(
    [Dim Produit].[Nom Produit].[Nom Produit].MEMBERS,
    10,
    [Measures].[Chiffre d'Affaires]
)
```

5. **Display Folder** : `Analyses Produits`

### Variante - Top 10 par Marge

6. **New Named Set**

7. **Name** : `[Top 10 Produits Marge]`

8. **Type** : **Dynamic**

9. **Expression** :

```mdx
TOPCOUNT(
    [Dim Produit].[Nom Produit].[Nom Produit].MEMBERS,
    10,
    [Measures].[Marge Commerciale]
)
```

10. **Display Folder** : `Analyses Produits`

### Test

11. **Browser** → Reconnect
12. **Glisser** `[Top 10 Produits CA]` dans les lignes
13. **Glisser** `Chiffre d'Affaires` dans les valeurs
14. **Ajouter un filtre** : Année = 2023
15. **Observer** : Le Top 10 change selon l'année sélectionnée (comportement dynamique)

### Test Contexte Multiple

16. **Nouvelle vue** :
    - Lignes : `[Top 10 Produits CA]`
    - Colonnes : `Type de Canal` → `Nom Canal`
    - Valeurs : `Chiffre d'Affaires`
    - Observer que le Top 10 s'adapte à chaque canal

## Partie 4 : Named Set 2 - Bottom 5 Produits (Dynamique)

### Contexte Métier

Identifier les produits à faible performance pour envisager des actions correctives ou leur retrait du catalogue.

### Création

1. **New Named Set**

2. **Name** : `[Bottom 5 Produits CA]`

3. **Type** : **Dynamic**

4. **Expression** :

```mdx
BOTTOMCOUNT(
    FILTER(
        [Dim Produit].[Nom Produit].[Nom Produit].MEMBERS,
        NOT ISEMPTY([Measures].[Chiffre d'Affaires])
    ),
    5,
    [Measures].[Chiffre d'Affaires]
)
```

5. **Display Folder** : `Analyses Produits`

**Note** : Le FILTER évite d'inclure les produits sans ventes.

### Variante - Produits Déficitaires

6. **New Named Set**

7. **Name** : `[Produits Marge Négative]`

8. **Type** : **Dynamic**

9. **Expression** :

```mdx
FILTER(
    [Dim Produit].[Nom Produit].[Nom Produit].MEMBERS,
    [Measures].[Taux de Marge] < 0
)
```

10. **Display Folder** : `Analyses Produits`

### Test

11. **Navigation** :
    - Lignes : `[Bottom 5 Produits CA]` et `[Produits Marge Négative]`
    - Valeurs : `CA`, `Marge Commerciale`, `Taux de Marge`
    - Analyser les produits à problème

## Partie 6 : Named Set 4 - Périodes Comparatives (Dynamique)

### Contexte Métier

Définir des périodes relatives pour faciliter les analyses de tendances.

### 6 Derniers Mois

1. **New Named Set**

2. **Name** : `[6 Derniers Mois]`

3. **Type** : **Dynamic**

4. **Expression** :

```mdx
LASTPERIODS(
    6,
    [Dim Temps].[Calendrier].[Mois Nom].MEMBERS.ITEM(
        [Dim Temps].[Calendrier].[Mois Nom].MEMBERS.COUNT - 1
    )
)
```

5. **Display Folder** : `Périodes`

### 12 Derniers Mois (Année Glissante)

6. **New Named Set**

7. **Name** : `[12 Derniers Mois]`

8. **Type** : **Dynamic**

9. **Expression** :

```mdx
LASTPERIODS(
    12,
    [Dim Temps].[Calendrier].[Mois Nom].MEMBERS.ITEM(
        [Dim Temps].[Calendrier].[Mois Nom].MEMBERS.COUNT - 1
    )
)
```

10. **Display Folder** : `Périodes`

### Année Courante vs Année Précédente

11. **New Named Set**

12. **Name** : `[Années Comparatives]`

13. **Type** : **Dynamic**

14. **Expression** :

```mdx
{
    [Dim Temps].[Calendrier].[Annee].MEMBERS.ITEM(
        [Dim Temps].[Calendrier].[Annee].MEMBERS.COUNT - 2
    ),
    [Dim Temps].[Calendrier].[Annee].MEMBERS.ITEM(
        [Dim Temps].[Calendrier].[Annee].MEMBERS.COUNT - 1
    )
}
```

15. **Display Folder** : `Périodes`

## Partie 10 : Named Set 8 - Analyses Avancées (Dynamique)

### Top 20% Produits (Principe de Pareto)

1. **New Named Set**

2. **Name** : `[Top 20% Produits - Pareto]`

3. **Type** : **Dynamic**

4. **Expression** :

```mdx
TOPCOUNT(
    [Dim Produit].[Nom Produit].[Nom Produit].MEMBERS,
    ROUND(
        [Dim Produit].[Nom Produit].[Nom Produit].MEMBERS.COUNT * 0.2,
        0
    ),
    [Measures].[Chiffre d'Affaires]
)
```

5. **Display Folder** : `Analyses Stratégiques`

### Produits en Croissance

6. **New Named Set**

7. **Name** : `[Produits Forte Croissance]`

8. **Type** : **Dynamic**

9. **Expression** :

```mdx
FILTER(
    [Dim Produit].[Nom Produit].[Nom Produit].MEMBERS,
    [Measures].[Croissance vs N-1 %] > 0.20
)
```

10. **Display Folder** : `Analyses Stratégiques`

### Clients à Risque (Baisse d'Activité)

11. **New Named Set**

12. **Name** : `[Clients Baisse Activité]`

13. **Type** : **Dynamic**

14. **Expression** :

```mdx
FILTER(
    [Dim Client].[Nom Client].[Nom Client].MEMBERS,
    [Measures].[Croissance vs N-1 %] < -0.30
    AND [Measures].[CA N-1] > 5000
)
```

15. **Display Folder** : `Analyses Clients`

### Test Pareto

16. **Vérifier le principe 80/20** :
    - Tableau 1 : `[Top 20% Produits - Pareto]` → CA
    - Tableau 2 : Tous les produits → CA Total
    - Calculer : CA Top 20% / CA Total
    - Devrait être proche de 80%

## Partie 12 : Tests Comparatifs Statique vs Dynamique

### Expérience 1 : Performance

1. **Créer deux sets identiques** :
   
   - `[Top 10 Static]` (Type: Static)
   - `[Top 10 Dynamic]` (Type: Dynamic)
   - Même expression MDX

2. **Activer SQL Server Profiler**

3. **Exécuter des requêtes** utilisant chaque set

4. **Comparer** :
   
   - Temps d'exécution
   - Nombre de requêtes au cube
   - Utilisation mémoire

### Expérience 2 : Adaptabilité

5. **Navigation avec Set Statique** :
   
   - Lignes : `[Top 10 Static]`
   - Filtre : Année 2023
   - Résultat : Liste fixe (calculée au traitement)

6. **Navigation avec Set Dynamique** :
   
   - Lignes : `[Top 10 Dynamic]`
   - Filtre : Année 2023
   - Résultat : Top 10 spécifique à 2023

7. **Changer le filtre** à 2024 :
   
   - Statique : Même liste
   - Dynamique : Liste adaptée à 2024

### Conclusions

**Utiliser Statique quand** :

- Liste connue et fixe
- Performance critique
- Peu de changements attendus

**Utiliser Dynamique quand** :

- Contexte variable
- Top N adaptatifs
- Analyses exploratoires

### Requêtes MDX pour Rapports

```mdx
-- Rapport 1 : Top Produits par Canal
SELECT 
    [Measures].[Chiffre d'Affaires] ON COLUMNS,
    [Top 10 Produits CA] ON ROWS,
    [Canaux Online] ON PAGES
FROM [Cube_Ventes]

-- Rapport 2 : Performance Géographique
SELECT 
    {
        [Measures].[Chiffre d'Affaires],
        [Measures].[Panier Moyen],
        [Measures].[Croissance vs N-1 %]
    } ON COLUMNS,
    [Europe Occidentale] ON ROWS

-- Rapport 3 : Analyse Clients Stratégiques
SELECT 
    {
        [Measures].[Chiffre d'Affaires],
        [Measures].[Nombre de Commandes],
        [Measures].[Panier Moyen]
    } ON COLUMNS,
    {
        [Clients VIP],
        [Clients Premium],
        [Clients VIP à Reconquérir]
    } ON ROWS
FROM [Cube_Ventes]

-- Rapport 4 : Dashboard Produits
SELECT 
    [Measures].[Chiffre d'Affaires] ON COLUMNS,
    {
        [Top 10 Produits CA],
        [Produits Forte Croissance],
        [Bottom 5 Produits CA]
    } ON ROWS
FROM [Cube_Ventes]
WHERE [Dim Temps].[Calendrier].[Annee].&[2024]
```
