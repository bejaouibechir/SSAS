# Créer un role dans ssdt tabulaire





1. **Ouvrez** le projet Tabular dans **SQL Server Data Tools (SSDT)**  
   → **Fichier > Ouvrir > Projet/Solution** → sélectionnez votre projet contenant le fichier .bim

2. Dans **Explorateur de solutions**, cliquez droit sur **Model.bim** → **Afficher le concepteur**

3. Dans la barre de menus, cliquez **Modèle > Gérer les rôles**  
   → la fenêtre **Gestionnaire de rôles** s’ouvre

4. Cliquez sur **Nouveau** pour créer un rôle

5. Saisissez un **Nom de rôle** (ex. : **ResponsableVentes**)  
   → (optionnel) ajoutez une **Description**

6. Sous l’onglet **Autorisations**, cochez **Lecture** (Read)  
   → ou **Lecture et traitement** si nécessaire

7. Passez à l’onglet **Membres** → cliquez **Ajouter**  
   → entrez les comptes (ex. : **DOMAINE\utilisateur** ou **utilisateur@entreprise.com**)

8. Pour la **sécurité au niveau des lignes** (RLS) :  
   → onglet **Filtres de lignes**  
   → sélectionnez une table (ex. : **FactInternetSales**)  
   → dans la zone de texte, écrivez une expression **DAX**  
     Exemple : `[SalesTerritoryKey] = 1`  
     ou `[DimGeography].[CountryRegionCode] = "FR"`

9. Cliquez **OK** pour enregistrer le rôle

10. **Testez** le rôle immédiatement :  
    → **Modèle > Analyser dans Excel**  
    → dans la fenêtre, choisissez le **rôle** que vous venez de créer  
    → cliquez OK → vérifiez que seules les données autorisées apparaissent dans le tableau croisé dynamique

11. **Déployez** le modèle pour utiliser le rôle en production :  
    → clic droit sur le projet dans l’Explorateur de solutions → **Déployer**  
    → connectez-vous au serveur **Analysis Services** → terminez le déploiement

Le rôle sera alors actif pour les utilisateurs/membres ajoutés une fois le modèle déployé.
